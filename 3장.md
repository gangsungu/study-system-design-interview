## 시스템 설계 면접 공략법
시스템 설계 면접은 설계 기술을 시연하는 자리이고 설계 과정에서 내린 결정들에 대한 방어능력을 보이는 자리

### 효과적 면접을 위한 4단계 접근법

1. 문제 이해 및 설계 범위 확정  
시스템 설계 면접에서는 생각없이 바로 답을 내서는 좋은 점수를 받기 어렵다  
요구사항을 완전히 이해하지 않고 답을 내놓는 행위는 아주 엄청난 부정적 신호다  

요구사항을 정확히 이해하는데 필요한 질문을 예시로 들어보면 아래와 같다  
+ 구체적으로 어떠한 기능을 만들어야 하나?
+ 제품의 사용자 수는 얼마나 되나?
+ 회사의 규모는 얼마나 빨리 커지리라 예상하나?
+ 회사가 주로 사용하는 기술스택은 무엇이고 기존 서비스에는 어떤 것이 있는가?

> 요구사항을 이해하고 모호함을 없애는 것이 이 단계에서 가장 중요하다

### 개략적인 설계안 제시 및 동의 구하기
설계안에 대한 최초 청사진을 제시하고 면접관에게 의견을 묻자, 핵심 컴포넌트를 그려서 설명하는 것도 좋다

### 상세 설계
해당 단계에서는 다음 목표를 달성한 상태일 것이다  

+ 시스템에서 전반적으로 달성해야 할 목표와 기능 범위 확인
+ 전체 설계의 개략적 청사진 마련
+ 해당 청사진에 대한 면접관의 의견 청취
+ 상세 설계에서 집중해야 할 영역들 확인

이제 면접관과 해야할 일은 설계 대상 컴포넌트 사이의 우선순위를 정하고 세부사항을 깊게 설명하는 것이다  
예를 들어 채팅 시스템에 관한 문제였다면 어떻게 하면 지연시간을 줄이고 사용자의 온/오프라인 상태를 표시할 것인지를 들어보고자 할 것이다

### 마무리
시스템의 전체적인 설계가 끝났다 이제 면접관이 몇가지 후속질문을 던질 차례인데, 주의해야 할 몇가지 부분이 있다

1. 면접관이 시스템 병목구간이나 좀 더 개선가능할 지점을 찾아내라 주문할 수 있다 이런 질문은 비판적 사고 능력을 보일 좋은 기회다  
2. 오류가 발생하면 어떤 일이 벌어지는지(서버 오류, 네트워크 장애 등) 따져보는 것도 좋다
3. 운영 이슈, 메트릭은 어떻게 수집하고 모니터링할 것인가, 시스템은 어떻게 배포할 것인가  
4. 미래에 닥칠 확장 규모에 어떻게 대처할 것인가

### 결론
+ 해야할 것
    - 질문을 통해 확인하자, 스스로 내린 가정이 옳다고 믿고 진행하지 말자
    - 문제의 요구사항을 이해하라
    - 정답이나 최선의 답안은 없다, 모든 결정의 트레이드 오프를 생각하자
    - 면접관이 우리의 사고 흐름을 이해할 수 있도록 소통하자
    - 가능하다면 여러 해법을 제시하는 것이 좋다
    - 개략적인 설계에 면접관이 동의하면 각 컴포넌트의 세부사항을 설명하자
    - 면접관의 아이디어를 이끌어내자 (내가 실제 회사에서 근무한다고 생각하고 면접에 임하자)
+ 주의할 점
    - 요구사항이나 가정을 분명히 하지 않은 상태에서 설계를 제시하지 말자
    - 처음부터 특정 컴포넌트의 세부사항을 깊게 설명하지 말자, 개략적인 설계를 마친 후 자세히 설계하자
    - 진행 중 막혔다면 힌트를 청해보자 (면접관과 소통을 두려워하지 말자)
